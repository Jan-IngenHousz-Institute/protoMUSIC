Phase 6 Plan: User Interaction & Feedback
=========================================

1. User-Accessible Reset/Re-Provision Command
---------------------------------------------
- Add CLI command (e.g., `wifi_reset` or `provision_reset`) to:
    - Clear WiFi credentials from NVS
    - Clear custom provisioning flag (namespace "wifi_prov", key "provisioned")
    - Trigger device reboot (optional)
- Add equivalent Lua function (e.g., `device.reset_provisioning()`)
- Document in CLI and Lua help
- Ensure DDD layering: CLI/Lua call into wifi_manager infrastructure

2. User Feedback for Provisioning Status/Failures
-------------------------------------------------
- Add status LED patterns for:
    - BLE provisioning active (e.g., slow blink)
    - Provisioning success (e.g., fast blink or solid)
    - Provisioning failure/timeout (e.g., error blink)
- Add CLI messages for:
    - Provisioning started, success, failure, timeout
- (Optional) Add Lua callback/event for provisioning result
- Ensure feedback is visible both during initial setup and on failures

3. Testing & Documentation
--------------------------
- Test reset and feedback features on hardware
- Update [docs/ble-provisioning.txt](ble-provisioning.txt) with reset/feedback usage
- Add troubleshooting section for common provisioning issues

---

**Note:**
- No changes to domain or device_commands layers required
- All new features are adapter/infrastructure only
- No forbidden dependencies introduced
