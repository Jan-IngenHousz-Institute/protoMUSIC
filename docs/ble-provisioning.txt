BLE Provisioning Reference
==========================

Device Config
-------------
  BLE name:  AMBYTE
  Security:  1 (X25519 + AES-CTR)
  PoP:       ambyte123
  Timeout:   5 minutes

Prerequisites (one-time)
------------------------
  pip install bleak protobuf cryptography

Provision via PC (Windows PowerShell)
-------------------------------------

$env:IDF_PATH = "$env:USERPROFILE\.platformio\packages\framework-espidf"

python "$env:IDF_PATH\tools\esp_prov\esp_prov.py" `
  --transport ble `
  --service_name "AMBYTE" `
  --sec_ver 1 `
  --pop "ambyte123" `
  --ssid "YOUR_WIFI_SSID" `
  --passphrase "YOUR_WIFI_PASSWORD"

Provision via Phone
-------------------
  Android: ESP BLE Provisioning (Play Store)
  iOS:     ESP BLE Provisioning (App Store)

  1. Scan for device "AMBYTE"
  2. Enter PoP: ambyte123
  3. Select WiFi network and enter password

Reset Provisioning
------------------
  Erase flash to clear NVS and force re-provisioning on next boot:

    pio run --target erase
    pio run --target upload

Notes
-----
  - Credentials are stored in NVS by esp_wifi and persist across reboots.
  - On subsequent boots, the device connects automatically using stored credentials.
  - BLE memory is freed after provisioning completes (WIFI_PROV_SCHEME_BLE_EVENT_HANDLER_FREE_BLE).
  - The custom NVS flag (namespace "wifi_prov", key "provisioned") tracks provisioning state.
